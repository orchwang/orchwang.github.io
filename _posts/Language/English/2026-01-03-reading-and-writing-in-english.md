## 들어가며

개발자로서 영어는 피할 수 없는 언어입니다. 공식 문서, Stack Overflow, GitHub 이슈, 기술 블로그 등 대부분의 기술 자료가 영어로 작성되어 있기 때문입니다. 이 시리즈에서는 **영어 읽기(Reading)**와 **쓰기(Writing)** 능력을 체계적으로 향상시키기 위한 학습 계획을 소개합니다.

## 학습 목표

### 읽기 (Reading)

- [ ] 기술 문서를 빠르고 정확하게 이해하기
- [ ] 영어 원서(기술 서적) 읽기
- [ ] 뉴스 및 아티클 읽기를 통한 어휘력 확장

### 쓰기 (Writing)

- [ ] 명확한 커밋 메시지 작성
- [ ] GitHub Issue / PR 설명 작성
- [ ] 기술 블로그 영어 포스트 작성
- [ ] 이메일 및 비즈니스 커뮤니케이션

## 학습 전략

### 1. 매일 읽기 습관 만들기

| 자료 유형   | 예시                    | 목표 시간 |
| ----------- | ----------------------- | --------- |
| 기술 문서   | MDN, Official Docs      | 30분      |
| 기술 블로그 | Medium, Dev.to          | 20분      |
| 뉴스        | Hacker News, TechCrunch | 10분      |

### 2. 쓰기 연습 루틴

```
Week 1-2: 짧은 문장 연습 (커밋 메시지, 코드 코멘트)
Week 3-4: 단락 연습 (PR 설명, Issue 작성)
Week 5-8: 긴 글 연습 (블로그 포스트, 문서화)
```

### 3. 어휘 확장 방법

- **기술 용어**: 문서를 읽으며 모르는 단어 정리
- **일반 어휘**: Anki 또는 Quizlet으로 반복 학습
- **문맥 학습**: 단어를 문장과 함께 암기

## 추천 리소스

### 읽기 자료

- [MDN Web Docs](https://developer.mozilla.org/) - 웹 기술 문서
- [Official Python Docs](https://docs.python.org/) - Python 공식 문서
- [Hacker News](https://news.ycombinator.com/) - 기술 뉴스

### 쓰기 도구

- **Grammarly** - 문법 검사 도구
- **Hemingway Editor** - 가독성 향상 도구
- **DeepL Write** - AI 기반 문장 개선

## 학습 로드맵

```
Phase 1 (1-2개월): 기초 다지기
├── 매일 기술 문서 읽기
├── 영어 커밋 메시지 작성
└── 기본 어휘 500개 학습

Phase 2 (3-4개월): 실력 향상
├── 영어 기술 서적 1권 완독
├── GitHub Issue/PR 영어로 작성
└── 중급 어휘 500개 학습

Phase 3 (5-6개월): 심화 학습
├── 영어 블로그 포스트 작성
├── 기술 문서 번역 또는 기여
└── 고급 표현 및 관용구 학습
```
